This guide will get you from zero to querying in just a few minutes. We'll cover the essential operations you need to start filtering and sorting your data with QueryKit.

## Prerequisites

Make sure you've [installed QueryKit](/installation) via NuGet:

```bash
dotnet add package QueryKit
```

## Basic Filtering

Let's start with a simple example. Say you have a `Person` entity:

```csharp
public class Person
{
    public Guid Id { get; set; }
    public string FirstName { get; set; }
    public string LastName { get; set; }
    public int Age { get; set; }
    public string Email { get; set; }
}
```

### Simple Filter

Apply a filter using the `ApplyQueryKitFilter` extension method:

```csharp
using QueryKit;

var filterInput = """Age > 18""";
var adults = _dbContext.People
    .ApplyQueryKitFilter(filterInput)
    .ToList();
```

### Multiple Conditions

Combine multiple conditions using `&&` (and) or `||` (or):

```csharp
var filterInput = """FirstName == \"Jane\" && Age > 18""";
var results = _dbContext.People
    .ApplyQueryKitFilter(filterInput)
    .ToList();
```

<Note>
  String and Guid values must be wrapped in double quotes. Use raw string literals (`"""..."""`)
  in C# 11+ for easier escaping.
</Note>

## Basic Sorting

Sorting is even simpler - just provide a comma-delimited list of properties:

```csharp
var sorted = _dbContext.People
    .ApplyQueryKitSort("LastName, FirstName desc")
    .ToList();
```

<Tip>
  Use `asc` or `desc` to specify sort direction. If not specified, QueryKit assumes ascending order.
  You can also use Sieve-style syntax with a `-` prefix for descending (e.g., `-Age`).
</Tip>

## Combining Filter and Sort

You can chain filtering and sorting operations:

```csharp
var filterInput = """Age >= 21""";
var results = _dbContext.People
    .ApplyQueryKitFilter(filterInput)
    .ApplyQueryKitSort("LastName, Age desc")
    .ToList();
```

Or use the combined `ApplyQueryKit` method:

```csharp
var results = _dbContext.People
    .ApplyQueryKit(new QueryKitData
    {
        Filters = """Age >= 21""",
        SortOrder = "LastName, Age desc"
    })
    .ToList();
```

## Common Operators

Here are the most commonly used operators to get you started:

### Comparison Operators

| Operator | Description | Example |
|----------|-------------|-------|
| `==` | Equals | `Age == 25` |
| `!=` | Not equals | `Status != \"Inactive\"` |
| `>` | Greater than | `Age > 18` |
| `<` | Less than | `Rating < 4.5` |
| `>=` | Greater than or equal | `Age >= 21` |
| `<=` | Less than or equal | `Age <= 65` |

### String Operators

| Operator | Description | Example |
|----------|-------------|-------|
| `@=` | Contains | `FirstName @= \"John\"` |
| `@=*` | Contains (case-insensitive) | `Email @=* \"gmail\"` |
| `_=` | Starts with | `LastName _= \"Sm\"` |
| `_=*` | Starts with (case-insensitive) | `FirstName _=* \"j\"` |
| `_-=` | Ends with | `Email _-= \".com\"` |
| `_-=*` | Ends with (case-insensitive) | `Email _-=* \".COM\"` |

<Tip>
  Append `*` to any string operator to make it case-insensitive!
</Tip>

### Logical Operators

```csharp
// AND - both conditions must be true
var input = """Age > 18 && Status == \"Active\"""";

// OR - at least one condition must be true  
var input = """FirstName == \"John\" || FirstName == \"Jane\"""";

// Parentheses for complex logic
var input = """(Age > 18 && Age < 65) || Status == \"Premium\"""";
```

## Working with Different Data Types

<Steps>
  <Step title="Strings and GUIDs">
    Wrap in double quotes:
    ```csharp
    var input = """FirstName == \"John\"""";
    var input = """Id == \"6d623e92-d2cf-4496-a2df-f49fa77328ee\"""";
    ```
  </Step>
  
  <Step title="Numbers">
    Use directly without quotes:
    ```csharp
    var input = """Age == 25""";
    var input = """Price > 99.99""";
    ```
  </Step>
  
  <Step title="Booleans">
    Must use explicit comparison:
    ```csharp
    var input = """IsActive == true""";
    var input = """IsDeleted == false""";
    ```
  </Step>
  
  <Step title="Dates and Times">
    Use ISO 8601 format with quotes:
    ```csharp
    var input = """Birthday == \"2022-07-01\"""";
    var input = """CreatedAt == \"2022-07-01T00:00:03Z\"""";
    var input = """Time == \"12:30:00\"""";
    ```
  </Step>
  
  <Step title="Null Values">
    No quotes needed:
    ```csharp
    var input = """MiddleName == null""";
    var input = """DeletedAt != null""";
    ```
  </Step>
</Steps>

## Property Aliases with Configuration

Customize property names for a better API experience:

```csharp
var filterInput = """first == \"Jane\" && age > 18""";

var config = new QueryKitConfiguration(config =>
{
    config.Property<Person>(x => x.FirstName).HasQueryName("first");
    config.Property<Person>(x => x.Age).HasQueryName("age");
});

var results = _dbContext.People
    .ApplyQueryKitFilter(filterInput, config)
    .ApplyQueryKitSort("first, age desc", config)
    .ToList();
```

## Real-World Example

Here's a complete example showing QueryKit in a typical Web API scenario:

```csharp
using Microsoft.AspNetCore.Mvc;
using QueryKit;

[ApiController]
[Route("api/[controller]")]
public class PeopleController : ControllerBase
{
    private readonly AppDbContext _dbContext;
    
    public PeopleController(AppDbContext dbContext)
    {
        _dbContext = dbContext;
    }
    
    [HttpGet]
    public async Task<IActionResult> GetPeople(
        [FromQuery] string? filters,
        [FromQuery] string? sortOrder)
    {
        var query = _dbContext.People.AsQueryable();
        
        // Apply filters if provided
        if (!string.IsNullOrWhiteSpace(filters))
        {
            query = query.ApplyQueryKitFilter(filters);
        }
        
        // Apply sorting if provided
        if (!string.IsNullOrWhiteSpace(sortOrder))
        {
            query = query.ApplyQueryKitSort(sortOrder);
        }
        
        var results = await query.ToListAsync();
        return Ok(results);
    }
}
```

Clients can now make requests like:

```bash
GET /api/people?filters=Age >= 21 && Status == "Active"&sortOrder=LastName, FirstName
```

## Error Handling

QueryKit throws specific exceptions for different error scenarios:

```csharp
using QueryKit.Exceptions;

try
{
    var results = _dbContext.People
        .ApplyQueryKitFilter(userInput)
        .ToList();
}
catch (ParsingException ex)
{
    // Invalid syntax or operators
    return BadRequest($"Invalid filter syntax: {ex.Message}");
}
catch (UnknownFilterPropertyException ex)
{
    // Property doesn't exist
    return BadRequest($"Unknown property: {ex.Message}");
}
catch (QueryKitException ex)
{
    // Base exception for all QueryKit errors
    return BadRequest($"Query error: {ex.Message}");
}
```
