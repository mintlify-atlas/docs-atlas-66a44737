QueryKit is a .NET library that makes it easier to query your data by providing a fluent and intuitive syntax for filtering and sorting. It's perfect for building flexible APIs where clients need to filter and sort data dynamically - think of it as a lighter weight alternative to OData or GraphQL for query operations.

## Why QueryKit?

QueryKit provides a simple, string-based syntax that translates directly into efficient LINQ expressions. No global configuration needed, no services to inject - just apply your filters and sorts exactly where you need them.

```csharp
var filterInput = """FirstName == \"Jane\" && Age > 10""";
var people = _dbContext.People
    .ApplyQueryKitFilter(filterInput)
    .ToList();
```

That's it! QueryKit automatically translates your filter string into an optimized expression that Entity Framework can convert to SQL.

## Filtering Capabilities

QueryKit supports a rich set of filtering operations:

- **Comparison Operators**: `==`, `!=`, `>`, `<`, `>=`, `<=`
- **String Operators**: `@=` (contains), `_=` (starts with), `_-=` (ends with)
- **Case-Insensitive**: Append `*` to any string operator (e.g., `@=*`, `==*`)
- **Logical Operators**: `&&` (and), `||` (or) with full parentheses support
- **Collection Filtering**: Query into nested collections with `Any` or `All` semantics
- **Property Grouping**: Search across multiple properties at once
- **Arithmetic Expressions**: Perform calculations directly in your queries
- **Custom Operations**: Define domain-specific filtering logic

```csharp
var input = """(FirstName, LastName) @=* \"john\" && Age >= 21 && Status == \"Active\"""";
var people = _dbContext.People
    .ApplyQueryKitFilter(input)
    .ToList();
```

## Sorting Capabilities

Sorting is straightforward with comma-delimited property lists:

```csharp
var people = _dbContext.People
    .ApplyQueryKitSort("LastName, FirstName desc")
    .ToList();
```

Supports:
- Multiple properties with independent sort directions
- `asc` or `desc` keywords (defaults to ascending)
- Sieve-style syntax with `-` prefix for descending
- Custom property aliases

## Configuration

Customize QueryKit's behavior with configuration options:

```csharp
var config = new QueryKitConfiguration(config =>
{
    // Custom property aliases
    config.Property<Person>(x => x.FirstName).HasQueryName("first");
    
    // Derived properties
    config.DerivedProperty<Person>(p => p.FirstName + " " + p.LastName)
        .HasQueryName("fullname");
    
    // Prevent filtering or sorting on specific properties
    config.Property<Person>(x => x.SocialSecurityNumber).PreventFilter();
    
    // Custom operators
    config.EqualsOperator = "eq";
    config.AndOperator = "and";
});

var people = _dbContext.People
    .ApplyQueryKitFilter("""first == \"Jane\"""", config)
    .ApplyQueryKitSort("first desc", config)
    .ToList();
```

## Use Cases

QueryKit is ideal for:

- **REST APIs** - Allow clients to specify filters and sorts via query parameters
- **Search Interfaces** - Build flexible search UIs with dynamic filtering
- **Data Tables** - Implement server-side filtering and sorting for grids
- **Admin Panels** - Provide powerful data exploration without custom endpoints
- **Reporting** - Let users customize report filters and ordering

## Inspired By

QueryKit is inspired by [Sieve](https://github.com/Biarity/Sieve) but with some key differences including more flexible syntax, enhanced operator support, property grouping, arithmetic expressions, and no global configuration requirements.
