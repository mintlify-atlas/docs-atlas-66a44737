## Overview

The `QueryKitConfiguration` class implements `IQueryKitConfiguration` and provides runtime configuration for QueryKit's filtering and sorting behavior. It allows you to customize operator syntax, property mappings, and parsing options.

This class is typically instantiated with a `QueryKitSettings` configuration action and provides immutable configuration at runtime.

**Namespace:** `QueryKit.Configuration`

## Constructor

### QueryKitConfiguration(Action\<QueryKitSettings\> configureSettings)

Creates a new configuration instance by applying settings through a configuration action.

```csharp
var config = new QueryKitConfiguration(settings =>
{
    settings.EqualsOperator = "eq";
    settings.AllowUnknownProperties = true;
    settings.MaxPropertyDepth = 3;
});
```

## Usage Example

```csharp
using QueryKit.Configuration;

// Create configuration with custom operators
var config = new QueryKitConfiguration(settings =>
{
    // Use SQL-style operators
    settings.EqualsOperator = "=";
    settings.NotEqualsOperator = "<>";
    
    // Allow filtering on unmapped properties
    settings.AllowUnknownProperties = true;
    
    // Limit nested property depth
    settings.MaxPropertyDepth = 3;
    
    // Configure property mappings
    settings.Property<User>(u => u.FirstName)
        .HasQueryName("name");
});

// Use configuration with QueryKit
var data = new QueryKitData
{
    Filters = "name = 'John'",
    Configuration = config
};

var results = dbContext.Users
    .ApplyQueryKit(data)
    .ToList();
```
