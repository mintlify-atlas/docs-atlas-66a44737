---
title: HasConversion Support
description: Use QueryKit with EF Core properties configured with HasConversion
---

For properties configured with EF Core's `HasConversion`, QueryKit provides special support that allows you to filter against the property directly without needing to access nested values.

## Configuration

Use the `HasConversion<TTarget>()` configuration method to enable support for properties with HasConversion:

```csharp
// EF configuration with HasConversion
builder.Property(x => x.Email)
    .HasConversion(x => x.Value, x => new EmailAddress(x))
    .HasColumnName("email");

// QueryKit configuration for HasConversion properties
var config = new QueryKitConfiguration(config =>
{
    config.Property<SpecialPerson>(x => x.Email)  // Point to Email property, NOT Email.Value
        .HasQueryName("email")
        .HasConversion<string>(); // Specify the target type used in HasConversion
});
```

## Usage

Once configured, you can filter directly against the property:

```csharp
var input = """email == "hello@gmail.com" """;
var people = _dbContext.People
    .ApplyQueryKitFilter(input, config)
    .ToList();
```

This allows you to use `Email == "value"` syntax instead of `Email.Value == "value"` when the property is configured with HasConversion in EF Core. The `HasConversion<TTarget>()` method tells QueryKit what the conversion target type is so it can handle the type conversion properly.

## HasConversion vs ComplexProperty/OwnsOne

<Warning>
  When using `HasConversion` in EF Core, you MUST configure the property in QueryKit using `x => x.Email`, not `x => x.Email.Value`. The conversion is on the parent property, so pointing to the nested `.Value` property will cause EF Core translation errors.
</Warning>

**Key Distinction:**

- **HasConversion**: Use `x => x.Email` in QueryKit (point to parent property)
- **ComplexProperty/OwnsOne**: Use `x => x.Email.Value` in QueryKit (point to nested property)

Use `x => x.Email.Value` only when using `ComplexProperty` or `OwnsOne` without HasConversion.
